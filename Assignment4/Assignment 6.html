In this assignment you will:
  • complete the stack class that you started work on in lab
  • write a queue class that implements the queue interface
  • write a program that uses your stack and queue classes to evaluate postfix expressions
Part 1 - complete the Stack class that you started in lab. 

Part 2 - write a Queue class.  Define ElementType as string. You may use a static array, a dynamic array or a linked list as the data structure.  If you use a dynamically allocated array or a linked list you will need to provide a destructor.  You do not need to provide an assignment operator or a copy constructor.  If you use an array your queue must be capable of holding up to capacity number of items and the push and pop operations must be O(1).  This means you will have to treat the array as if it were circular (as discussed in class on Thursday). 

Part 3 - write a well decomposed program that:
  • for each line of an input file
    • Reads a postfix expression from an input file (the program user must be able to enter the name of the file to be read from) and stores the tokens that make up the expression in a queue of tokens (strings). 
    • Evaluates the postfix expression.  The function that does the evaluation (algorithm is given below and in the Unit 4 slides) will make use of a stack of operands (ints).  
    • Writes to a text file the postfix expression and the result of evaluating it.
You may find the atoi() function from <cstdlib> useful.

Algorithm to evaluate a postfix expression

create an empty stack
while there are more tokens in the postfix expression
        get the next token
        if the token is an operand
                push it onto the stack
        if the token is an operator
                get and pop the top two operands from the stack
                apply the operator to these two operands
                push the result onto the stack
 the top of the stack is the result of evaluating the postfix expression

The text file that contains the postfix expressions will have the following format.  Each line will contain one postfix expression.   All operands with be integers and the only operators will be +, * and -.   Tokens (operators and operands) will be separated by at least one blank and the character # will mark the end of a postfix expression.  An example of an input file follows:

            23 4 * 19 2 - + #
            6 3 -  #
            36 #

You can assume that the input file is formatted as described above and that all infix expressions are legal. You can assume that the last character on each line will be a #. 

The output file your program creates as a result of the reading the input file shown above should look something like:

The result of evaluating  23 4 * 19 2 - +   is 109
The result of evaluating 6 3 - is 3
The result of evaluating 36 is 36
